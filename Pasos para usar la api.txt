Paso número 5: El proyecto necesita conectarse a su base de datos local. Para eso, deben crear un archivo de configuración:
-En la raíz del proyecto (petcare-services), busquen el archivo llamado .env.example.
-Creen una copia de este archivo y renómbrenla a .env (es muy importante que se llame exactamente así).
-Abran el archivo .env y modifiquen los valores de las variables de entorno para que coincidan con la configuración de su computadora:
**PGHOST: Es la dirección de su servidor de base de datos. Normalmente es localhost.
**PGUSER: El nombre de usuario de PostgreSQL (el más común es postgres).
**PGPASSWORD: La contraseña que configuraron al instalar PostgreSQL.
**PGDATABASE: El nombre de la base de datos que van a crear en el siguiente paso.
**PGPORT: El puerto de PostgreSQL (casi siempre es 5432)
**PORT: El puerto en el que correrá la API (por defecto es 3000)

Paso numero 6: En caso de q no les deje clonar la base de datos en services se encuentra el query con los datos y ejecutalo en postgres (Necesitan crear primero una base de datos (ej: petcare_db) y luego ejecutar el query ahí) 

Nota: la base de datos esta hecha en postgres, si no lo tiene descargenlo

paso numero 7: Asegúrate de estar dentro de la carpeta petcare-services en tu terminal y ejecuta el comando: npm install

paso 8: cuando vayas a utilizar la api es decir siempre q vayas a registrar debes usar el comando npm start en la terminal de vs code, si sale un OK en la pantalla entonces esta listo y corriendo, si no mandame un mensaje y vamos a verificar que hacer falta

(Los cambios que hayas hecho en la env.example no los subas, esos son locales solo para ti no universales)
